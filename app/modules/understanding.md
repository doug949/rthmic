# PILLAR: Understanding

## USER INTENT
The user wants to understand a concept well enough to explain it simply.

## CORE FUNCTION
Create an onramp from confusion into clarity using acknowledgement, a simple mental model, concrete examples, and a summary loop.

## EMOTIONAL STATE
Confused or blocked -> clear, oriented, and able to explain

## SONG BEHAVIOUR
- Structure: acknowledge confusion, offer a mental model, break into 3-5 key parts, illustrate with examples, close with a summary loop.
- Tone: patient, plain, and reassuring.
- Language: concrete, simple, and uncluttered.
- Musical feel: clear melodic phrasing with repeated summary lines.

## STRUCTURAL RULES
- Start by acknowledging that confusion is present.
- Provide one simple mental model before details.
- Break the concept into 3-5 key parts.
- Use concrete examples to make each part graspable.
- End with a summary loop the user can repeat.

## LANGUAGE RULES
- Use everyday wording.
- Prefer one strong example over many weak ones.
- Keep each idea distinct.
- Use repetition to reinforce the mental model.
- Make the final loop feel like a clear takeaway.

## CONSTRAINTS
- Do not over-explain.
- Do not include edge cases.
- Do not aim for completeness.
- Do not use technical language unless the concept requires it.
- Do not mix in memory chaining, menu options, rescue pacing, or movement loops.

## SUCCESS CRITERIA
The user feels able to explain the concept simply in their own words.
